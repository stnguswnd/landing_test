# PRD — 영어 학원 단일형 랜딩 페이지

## 1. 문서 개요
- **프로젝트명**: Green English Academy Landing Page
- **문서 목적**: 영어 학원용 단일 페이지 랜딩 사이트의 제품 요구사항 정의
- **구현 스택 가정**: Next.js(App Router), TypeScript, Tailwind CSS, Supabase(추후 게시판/음원용), Vercel 배포
- **참고 방향**:
  - 상단 글로벌 메뉴를 제거하고, 한 페이지 내 스크롤 중심 구조로 재구성
  - 섹션 구성은 `오버뷰 / 브랜드 / 통합 로드맵 / 학습 시스템 / 입학 절차 / 상담 CTA` 중심
  - 톤앤매너는 기존 레퍼런스의 정보형 랜딩 구성을 유지하되, **그린 계열 브랜딩**으로 재해석

---

## 2. 제품 목표

### 2.1 핵심 목표
1. 학부모/학생이 첫 방문 시 학원의 차별점을 10초 안에 이해할 수 있어야 한다.
2. 스크롤만으로 학원 소개, 커리큘럼, 로드맵, 시스템, 입학 프로세스를 자연스럽게 파악할 수 있어야 한다.
3. 최종적으로는 **상담 신청/레벨 테스트 신청/문의 전환**으로 이어져야 한다.
4. SEO를 고려해 검색 유입이 가능한 구조여야 한다.
5. 추후 게시판, 공지, 음원(mp3) 플레이 기능이 추가되어도 구조가 무너지지 않아야 한다.

### 2.2 비즈니스 목표
- 브랜드 신뢰 확보
- 설명회/상담 예약 유도
- 레벨 테스트 신청 전환
- 지역 기반 학원 검색 유입 확보
- 향후 콘텐츠 마케팅(공지사항, 학습 자료, 음원) 확장 기반 확보

---

## 3. 타겟 사용자

### 3.1 1차 타겟
- 초등~중등 자녀를 둔 학부모
- 영어 실력 향상뿐 아니라 입시/서술형/문해력/말하기까지 함께 보는 학원을 찾는 사용자

### 3.2 2차 타겟
- 중학생/고등학생 본인
- 귀국/국제학교/특목 대비 등 특정 목적이 있는 학습자
- 학원 브랜드를 비교 탐색하는 사용자

### 3.3 사용자 니즈
- “이 학원이 무엇이 다른가?”를 빠르게 이해하고 싶다.
- “우리 아이가 어디에 해당되는지”를 쉽게 알고 싶다.
- “수업 방식과 학습 관리가 체계적인지”를 보고 싶다.
- “입학 절차가 복잡하지 않은지”를 알고 싶다.
- 모바일에서도 읽기 쉬워야 한다.

---

## 4. 핵심 가치 제안
- **문해력 + 사고력 + 실전 영어**를 함께 설명하는 구조
- 연령/레벨/목표에 따른 **통합 로드맵** 제공
- 오프라인 수업 + 디지털 학습 + 성과 관리의 **학습 시스템화**
- 복잡한 설명 대신 **시각적 블록 중심** 정보 전달
- 마지막 CTA까지 자연스럽게 이어지는 **퍼널형 랜딩 구조**

## 6. 섹션별 요구사항

## 6.1 Hero
### 목적
- 첫 화면에서 브랜드 정체성과 핵심 메시지 전달
- CTA 클릭 유도

### 포함 요소
- 메인 카피
- 서브 카피
- 핵심 가치 2~3개 태그
- 대표 CTA 2개
  - 상담 신청
  - 레벨 테스트 예약
- 시각 요소
  - 그린 브랜딩 중심의 일러스트/사진/도형 조합
  - 과한 애니메이션보다 신뢰감 있는 정적/약한 모션 사용

### 예시 메시지 방향
- “문해력과 사고력을 함께 키우는 영어 학습”
- “레벨별 맞춤 로드맵으로 흔들리지 않는 영어 성장”

---

## 6.2 Overview
### 목적
- 교육 시장 변화와 학원의 필요성을 짧게 설명

### 포함 요소
- 왜 지금 이 영어 교육이 필요한지
- 기존 영어교육과의 차이점 비교 카드
- 핵심 포인트 3개 내외
  - 문해력
  - 사고력
  - 실전 적용

### UI 권장
- 3열 카드 또는 2x2 비교 카드
- 숫자/아이콘/짧은 설명 조합

---

## 6.3 Brand
### 목적
- 학원의 철학과 브랜드 차별점 전달

### 포함 요소
- 브랜드 소개 문장
- 교육 철학 3개
- 강사/운영/관리 철학
- 필요 시 “왜 믿을 수 있는가” 블록 추가

### 권장 블록
- Mission
- Teaching Philosophy
- Student Growth Promise

---

## 6.4 Integrated Roadmap
### 목적
- 레벨/연령/목표별 경로를 한눈에 보여줌

### 포함 요소
- 연령별 로드맵
- 레벨별 로드맵
- 목표별 로드맵
  - 내신
  - 서술형
  - 특목/국제
  - 수능/심화

### UI 권장
- 탭 또는 카드 전환형
- 데스크탑에서는 표/로드맵 바 형태
- 모바일에서는 세로 카드 스택 형태

### 사용자 경험 포인트
- “우리 아이는 어디쯤인가?”에 바로 답할 수 있어야 함

## 6.5 Learning System
### 목적
- 수업 방식과 학습 관리 시스템의 신뢰도 확보

### 포함 요소
- 수업 구조

- 온라인 연계 학습

- 추후 확장 기능 예고

  - mp3 음원 플레이어

### 세부 구성 예시

2. 오프라인 수업

4. 디지털 학습/음원 청취
5. 평가/피드백
6. 학부모 확인

### UI 권장
- Step flow
- 5~6단 프로세스 카드
- 앱/학습 화면 mockup 섹션

## 9. SEO 요구사항
- Next.js App Router 기반 SSR/SSG 활용
- 각 섹션의 의미 있는 heading 구조 사용(H1~H3)
- 브랜드/지역/서비스 키워드 반영
- 메타 타이틀/디스크립션/OG 이미지 제공
- schema.org 구조화 데이터 적용 권장
  - EducationalOrganization
  - FAQPage
  - BreadcrumbList(필수는 아님)
- URL slug 예시
  - `/`
  - 추후 `/notice`, `/media`, `/faq`

### 예시 SEO 키워드
- 지역명 + 영어학원
- 초등 영어학원
- 중등 영어학원
- 문해력 영어
- 서술형 영어
- 영어 레벨테스트
